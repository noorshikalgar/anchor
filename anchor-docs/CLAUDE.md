# Anchor — instructions for Claude Code

This repo currently contains **planning only** (see `README.md` and `docs/`).
There is no code yet — you are building it from scratch.

Before writing any code:
1. Read `README.md`.
2. Read all 12 files in `docs/` in numeric order. They are short by design —
   this should take a few minutes and will save far more time than it costs.
3. Check for a `frontend-design` skill or equivalent UI/UX guidance available
   in your environment and load it before building any screens — the design
   system doc (`docs/08-design-system.md`) assumes that guidance as a base
   layer and builds a specific direction on top of it.

Then follow `docs/12-roadmap-mvp.md` and build **Phase 0 only** in your first
session. Stop and let the person react before continuing to later phases —
don't build the AI layer (Phase 2) before the core logging loop (Phase 0)
is solid and has been tried by hand.

If any instruction in this repo's docs seems to conflict with something the
person asks for directly in chat, the person's live instruction wins — these
docs are a starting brief, not a contract.
